# IPoEd

## 概要

IPoEd は NTT 東日本・NTT 西日本のフレッツ光 (コラボモデルを含む) における IPoE 方式のインターネット接続サービスを利用したルータを構築するためのソフトウェアです。
Ubuntu での利用を前提としています。

IPoEd は以下の機能を提供します。

* WAN 側で IPv6 プレフィクスが配布される方式が ICMPv6 RA か DHCPv6 PD かを自動で判別して LAN 側に IPv6 アドレスを設定する
  * DHCPv6 PD 方式の回線に HGW を接続し、その LAN 側で IPoEd が稼働している場合、DHCPv6 PD によるプレフィクスの再委譲は受けず、ICMPv6 RA 方式としてアドレス設定を行います
* ICMPv6 RA 方式の場合:
  * WAN 側から受け取った RA の IPv6 プレフィクスを LAN 側に RA で再配布する機能
  * LAN 側からの ICMPv6 NA をキャッシュし WAN 側からの ICMPv6 NS に代理で応答する機能
* DHCPv6 PD 方式の場合:
  * WAN 側から受け取った PD の IPv6 プレフィクスの先頭サブネットを LAN 側に RA で配布する機能
* WAN 側から受け取った DNS キャッシュサーバ一覧と DNS 検索一覧を LAN 側に DHCPv6 Information-Request で配布する機能
* HB46PP 第 1.2 版(※1) による IPv4 over IPv6 プロビジョニングクライアント機能(ただし、対応するマイグレーション技術は IPIP トンネル接続と DS-Lite のみ)
* IPv6/IPv4 の接続性に変化があった際にスクリプトを実行する機能(フック機能)

IPoEd は以下の機能は提供しません。

* パケットのフォワーディング・ブリッジ
  * パケットの転送は Linux カーネルが担います
* IPv4 DHCP サーバ機能
* IPv4 DNS フォワーダ機能

(以下の「使い方」では IPv4 DHCP サーバ機能と DNS フォワーダ機能をそれぞれ `isc-dhcp-server` と `pdns-recursor` で設定する方法を説明します。)

2026/04/06 時点で Ubuntu 24.04 LTS で動作検証しています。

2026/04/20 時点で HB46PP は OCX光 インターネット(※2)または v6IX(※3) によるインターネット接続サービスの ICMPv6 RA モードと DHCPv6 PD モードで動作検証しています。

* (※1) https://github.com/v6pc/v6mig-prov/blob/1.2/spec.md
* (※2) https://ocx-hikari.net
* (※3) https://ocx-hikari.net/v6ix

## 使い方

`ipoed_0.1.0-1_amd64.deb` をインストールします。

```shell
$ sudo dpkg -i ipoed_0.1.0-1_amd64.deb
Selecting previously unselected package ipoed.
(Reading database ... 71954 files and directories currently installed.)
Preparing to unpack ipoed_0.1.0-1_amd64.deb ...
Unpacking ipoed (0.1.0-1) ...
Setting up ipoed (0.1.0-1) ...
ipoed.service is a disabled or a static unit, not starting it.
```

`/etc/default/ipoed` を編集します。
最低限 `WAN_IF` と `LAN_IF` は自分の環境のインターフェース名に置き換える必要があります。

```shell
$ sudo vi /etc/default/ipoed
WAN_IF="enp1s0"
LAN_IF="enp2s0"
IPOED_OPTS="--lan-addr4 192.168.1.1/24"
```

> [!WARNING]
> `WAN_IF` と `LAN_IF` に指定したインターフェースは Netplan で設定を行わないようにする必要があります。
> Netplan の設定からは削除してください。
> Netplan の設定に `WAN_IF` と `LAN_IF` に指定したインターフェースがあると IPoEd が設定したアドレスが Netplan によって変更される可能性があります。
> 
> また、IPoEd 自身はシステムのリゾルバを用いるため、名前解決の際には `/etc/resolv.conf` に設定された DNS キャッシュサーバ (通常は `systemd-resolved`) に問い合わせが行われます。
> しかし、IPoE 回線で IPv4 over IPv6 を HB46PP によりプロビジョニングする場合、
> 特定のドメイン名のレコード (具体的には `4over6.info. IN TXT`) を回線から DHCPv6 で得られる DNS キャッシュサーバに問い合わせて、
> プロビジョニングに必要な情報を取得します。
> そのため、例えば Netplan の設定で `WAN_IF` や `LAN_IF` 以外のインターフェースで IPv4 の DHCP が設定されていて、DHCP サーバから DNS キャッシュサーバの情報を得た場合など、
> `systemd-resolved` が回線から得られる DNS キャッシュサーバ以外に問い合わせを行う状態では、プロビジョニングに必要な情報が得られず、
> HB46PP による IPv4 over IPv6 の設定に失敗します。

以降の例では `WAN_IF` が `enp1s0` で `LAN_IF` が `enp2s0` であるとして説明します。
それぞれ実際のインターフェースに置き換えてください。

`IPOED_OPTS` には以下の設定をすることができます。

```shell
    -i, --lan-iid <LAN_IID>        LAN IID (Default: SLAAC)
    -s, --dns-server-self          Set DHCPv6 DNS server self
    -4, --lan-addr4 <LAN_ADDR4>    LAN IPv4 address (Default none)
    -H, --disable-hb46pp           Disable HB46PP
    -u, --hb46pp-user <USER>       HB46PP user
    -p, --hb46pp-pass <PASS>       HB46PP pass
    -e, --hook-path <HOOK_PATH>    Hook path
    -d, --debug                    Enable debug
```

LAN 側に設定する IPv4 アドレスを変えたいときは `--lan-addr4` の IPv4 アドレスを書き換えます。

LAN 側に設定する IPv6 アドレスの IID (インターフェース ID) をデフォルトの SLAAC (EUI-64) から変更する場合は以下の例のように `IPOED_OPTS` に `--lan-iid` を追加します。

```shell
IPOED_OPTS="--lan-addr4 192.168.1.1/24 --lan-iid ::1"
```

IPoEd はデフォルトでは WAN 側から取得した DNS キャッシュサーバ一覧と DNS 検索一覧をそのまま LAN 側に DHCPv6 で配布します。
もし LAN 側で ActiveDirectory が動作しているなどの理由から直接 DNS キャッシュサーバを設定したくないときは `--dns-server-self` を設定することで自身の LAN 側の IPv6 アドレスを DNS キャッシュサーバとして配布します。
(その上で `pdns-recursor` で ActiveDirectory のゾーンを ActiveDirectory サーバへ向けるなどしてください。)

IPv4 を利用しないときや HB46PP による IPv4 over IPv6 プロビジョニングが利用できないときは `--disable-hb46pp` を設定することで HB46PP の接続を試みる動作を止めることができます。

IPoEd は上流から DHCPv6 でもらった DNS キャッシュサーバの情報を `/etc/systemd/resolved.conf.d/ipoed.conf` に書き込み `systemctl restart systemd-resolved.service` を実行します。
Ubuntu Linux では通常 `/etc/systemd/resolved.conf.d` が存在しないので作成しておきます。

```shell
$ sudo mkdir /etc/systemd/resolved.conf.d
```

Linux で IPv4 と IPv6 のパケットをフォワーディングするためには `sysctl` で `net.ipv4.ip_forward` と `net.ipv6.conf.all.forwarding` と `net.ipv6.conf.default.forwarding` を `1` に設定する必要があります。
以下のような内容のファイルを作成し IPv4 と IPv6 のフォワーディングができるようにします。
(反映する一番簡単な方法は再起動することです。)

```shell
$ sudo vi /etc/sysctl.d/ipoed.conf
net.ipv4.ip_forward=1
net.ipv6.conf.all.forwarding=1
net.ipv6.conf.default.forwarding=1
```

IPv4 は NAPT ができるように、IPv6 は外からのアクセスをブロックするように、それぞれ `iptables` と `ip6tables` を設定します。

再起動で自動的に設定が反映されるよう `iptables-persistent` をインストールします。

```shell
$ sudo apt install iptables-persistent
```

(`enp2s0` は実際の `LAN_IF` に置き換えてください。)

IPv4 の端末のために DHCP サーバとして `isc-dhcp-server` をインストールします。

```shell
$ sudo apt install isc-dhcp-server
```

`/etc/dhcp/dhcpd.conf` の末尾に以下のような設定を追記します。
(`--lan-addr4` で LAN 側の IPv4 アドレスを変更したときは適宜修正してください。)

```shell
$ sudo vi /etc/dhcp/dhcpd.conf
shared-network DHCP192168001 {
    subnet 192.168.1.0 netmask 255.255.255.0 {
        option domain-name-servers 192.168.1.1;
        option routers 192.168.1.1;
        default-lease-time 86400;
        max-lease-time 86400;
        range 192.168.1.100 192.168.1.199;
    }
    on commit {
        set shared-networkname = "DHCP192168001";
    }
}
```

`isc-dhcp-server` を起動時に自動起動するよう設定しサービスを起動します。

```shell
$ sudo systemctl enable isc-dhcp-server
Synchronizing state of isc-dhcp-server.service with SysV service script with /usr/lib/systemd/systemd-sysv-install.
Executing: /usr/lib/systemd/systemd-sysv-install enable isc-dhcp-server
$ sudo systemctl start isc-dhcp-server
```

IPv4 の端末のために DNS フォワーダとして `pdns-recursor` をインストールします。

```shell
$ sudo apt install pdns-recursor
```

`/etc/powerdns/recursor.d/ipoed.conf` として以下のような内容のファイルを作成します。
(`--lan-addr4` で LAN 側の IPv4 アドレスを変更したときは適宜修正してください。)


```shell
$ sudo vi /etc/powerdns/recursor.d/ipoed.conf
local-address=192.168.1.1
```

`pdns-recursor` を起動時に自動起動するよう設定しサービスを起動します。

```shell
$ sudo systemctl enable pdns-recursor
$ sudo systemctl start pdns-recursor
```

IPoEd が起動時に自動起動するように設定しサービスを起動します。

```shell
$ sudo systemctl enable ipoed
$ sudo systemctl start ipoed
```

## フック機能

`/usr/libexec/ipoed/hook` というパスで実行可能なスクリプトを置いておくことで IPv6 のアドレスが設定(変更含む)されたとき、 IPv6 アドレスが削除されたとき、 IPv4 が接続されたとき、 IPv4 が切断されたときに実行されます。

スクリプトのパスは `/etc/default/ipoed` の `IPOED_OPTS` を以下のように設定することで変更することもできます。

```shell
IPOED_OPTS="--lan-addr4 192.168.1.1/24 --hook-path /path/to/hook"
```

実行時、以下の環境変数が設定されます。

* `IPOED_HOOK_EVENT` ... イベントの種類を表します。 IPv6 のアドレスが設定されたときは `IPV6_UP` が、 IPv6 のアドレスが削除されたときは `IPV6_DOWN` が、 IPv4 が接続されたときは `IPV4_UP` が、 IPv4 が切断されたときは `IPV4_DOWN` が、それぞれ設定されます。
* `IPOED_WAN_IF` ... `WAN_IF` の値が設定されます。
* `IPOED_LAN_IF` ... `LAN_IF` の値が設定されます。
* `IPOED_IPV4_ADDR` ... HB46PP の結果が IPIP トンネルで IPv4 アドレスが設定されたときはその値(例: `203.0.113.123`)が埋まります。
* `IPOED_IPV6_ADDR` ... `LAN_IF` に設定した IPv6 アドレス(例: `2001:db8:1234:5600:2e2:69ff:fe5f:8266`)が埋まります。

## 補足

* ICMPv6 RA モードで動作する場合は IPoEd が自動で `WAN_IF` をプロミスキャスに設定します。これは WAN 側から送られてくる ICMPv6 NS の宛先アドレスがターゲットの IPv6 アドレスに対応するマルチキャストアドレスとなりますが LAN 側の全ての IPv6 アドレスに対応するマルチキャストグループにあらかじめ入っておくことが困難なためです。`WAN_IF` をプロミスキャスにすることで全ての ICMPv6 NS を拾えるようになります。`ip link set promisc off dev $WAN_IF` したりすると LAN 側の端末が IPv6 で通信できなくなるのでご注意ください。状態が切断された際は自動でプロミスキャスを無効にします。
* ICMPv6 RA モードでは ONU/HGW と WAN 側の間に端末を設置することはできません。 WAN 側の外側の端末から LAN 側への通信は(`ip6tables` で許可されていれば)可能ですが、 LAN 側から WAN 側の外側に設置された端末への通信は(LAN 側から送られた ICMPv6 NS が WAN 側の外側へ転送されないため)できません。
* フックは前の実行が完了してから次の実行が開始されます。例えば `IPV6_DOWN` と `IPV6_UP` がほぼ同時に起こるような状況があった場合 `IPV6_DOWN` の分の実行が完了してから `IPV6_UP` の分の実行が開始されます。フックの実行は IPoEd 側ではタイムアウト処理を行わないため必ず完了するように注意する必要があります。フックが完了しなかった場合、それ以降のフックが実行されなくなります。
